# .deb Signature Verification Flow

## Overview
This document traces the complete flow of `.deb` GPG signature verification in APM.

## Configuration
**sources.list syntax:**
```
deb [deb-signatures=required|optional|disabled] https://example.com/debian bookworm main
```

## Installation Flow

### 1. Package Resolution (`installWithDeps`)
- Resolves dependencies
- Downloads `.deb` files
- For each package:
  - ✅ Verifies SHA256/MD5 checksums (existing)
  - ✅ **NEW:** Verifies GPG signatures if policy enabled

### 2. Policy Determination (`getDebSigPolicy`)
```cpp
DebSignaturePolicy getDebSigPolicy(repoIndices, pkg)
  for each repo index:
    if matches pkg.repoUri and pkg.repoDist:
      return index.source.debSignaturePolicy
  return Disabled
```

### 3. Signature Verification Logic
```
IF policy != Disabled:
  debSha = SHA256(debPath)
  
  IF debSha in cache:
    ✅ verified (skip verification)
  ELSE:
    sigPath = ensureDebSignature(pkg, debPath)
    
    IF sigPath not found:
      IF policy == Required:
        ❌ ABORT install
      ELSE:
        ⚠️ WARN and continue
    ELSE:
      verified = verifyDebGpg(pkg, debPath, sigPath, &fingerprint)
      
      IF verified:
        cache[debSha] = {sigType, sigSource, sigPath, verifiedBy: fingerprint}
        storeSigCache()
        ✅ continue
      ELSE IF policy == Required:
        ❌ ABORT install
      ELSE:
        ⚠️ WARN and continue
```

### 4. Signature Download (`ensureDebSignature`)
```cpp
// Check local cache first
IF exists(debPath + ".asc"):
  return debPath + ".asc"
IF exists(debPath + ".gpg"):
  return debPath + ".gpg"

// Download from repo
sigUrl = makeSigUrl(pkg, ".asc")  // prefer .asc
IF downloadFile(sigUrl, debPath + ".asc"):
  return debPath + ".asc"

sigUrl = makeSigUrl(pkg, ".gpg")  // fallback .gpg
IF downloadFile(sigUrl, debPath + ".gpg"):
  return debPath + ".gpg"

return error "No detached signature available"
```

### 5. GPG Verification (`verifyDebGpg` → `verifyDetachedSignature`)
```cpp
verifyDetachedSignature(debPath, sigPath, trustedKeysDir, &err, &fingerprintOut)
  - Parse OpenPGP signature packet
  - Require RSA + SHA256
  - Hash the .deb file
  - For each trusted key in trustedKeysDir:
    - Try RSA PKCS#1 v1.5 verification
    - IF match:
      fingerprintOut = key fingerprint
      return true
  return false
```

### 6. Cache Storage
**Location:** `<PKGS_DIR>/sig-cache.json`

**Format:**
```json
{
  "<deb-sha256-hex>": {
    "sigType": "asc",
    "sigSource": "repo",
    "sigPath": "/data/apm/pkgs/package.deb.asc",
    "verifiedBy": "<key-fingerprint-sha256>"
  }
}
```

## Verification Points

### ✅ Correctness Checklist
1. ✅ Policy parsed from sources.list
2. ✅ Policy matched to package by repo URI + dist
3. ✅ Signature downloaded (.asc preferred, .gpg fallback)
4. ✅ Signature stored next to .deb in PKGS_DIR
5. ✅ GPG verification uses existing RSA/SHA256 logic
6. ✅ Fingerprint extracted and cached
7. ✅ Cache keyed by .deb SHA256 (content-addressable)
8. ✅ Cache persisted as JSON
9. ✅ Required policy aborts on failure
10. ✅ Optional policy warns and continues
11. ✅ Disabled policy skips entirely
12. ✅ Cache hit skips re-verification

## Testing Strategy

### Manual Testing
```bash
# 1. Add trusted key
apm key-add /path/to/repo-key.asc

# 2. Configure source with required signatures
echo 'deb [deb-signatures=required] https://repo.example.com/debian bookworm main' > /data/apm/sources/sources.list

# 3. Update indices
apm update

# 4. Install package (should verify signature)
apm install <package>

# 5. Check cache
apm sig-cache show

# 6. Reinstall (should use cache)
apm install --reinstall <package>

# 7. Clear cache and reinstall
apm sig-cache clear
apm install --reinstall <package>
```

### Error Cases
- Missing signature (required) → Install aborts
- Invalid signature (required) → Install aborts
- Missing signature (optional) → Warning, install continues
- Invalid signature (optional) → Warning, install continues
- No trusted keys → Verification fails
- Wrong key → Verification fails

## Security Properties

1. **Trust Chain:**
   - Release signature verified via trusted keys
   - Packages SHA256 verified from Release
   - .deb SHA256/MD5 verified from Packages
   - **NEW:** .deb GPG verified via trusted keys

2. **Cache Safety:**
   - Keyed by .deb SHA256 (content changes = new verification)
   - Stores fingerprint for audit trail
   - Not a security vulnerability if cache is tampered (just re-verifies)

3. **Policy Enforcement:**
   - Required = Strong security (mandatory verification)
   - Optional = Best-effort (verify if available)
   - Disabled = Current behavior (checksums only)

## Known Limitations

1. Only RSA + SHA256 signatures supported
2. No revocation checking (rely on key removal)
3. Signature availability depends on repo configuration
4. Cache has no TTL (infinite unless cleared)
