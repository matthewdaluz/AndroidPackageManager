APM Termux Compatibility Layer — Technical Specification

**Version:** 1.0
**Author:** Matthew DaLuz & ChatGPT
**Scope:** This document defines all required architectural changes for implementing full Termux package support inside APM using an APM-owned Termux root.

---

## **1. Overview**

Termux packages are not traditional Debian packages.
They require a Termux runtime environment, filesystem hierarchy, and unique dependency model.

The goal of this specification is to enable APM to:

1. Detect Termux repositories
2. Detect Termux packages
3. Extract Termux packages into a dedicated APM-owned Termux root
4. Provide a functional Termux-like runtime environment
5. Wrap Termux binaries so they run using APM’s Termux environment
6. Resolve dependencies using a Termux-only logic path
7. Keep Debian-style and Termux-style packages completely separate
8. Handle install, upgrade, and removal correctly

No modifications to the real Termux app installation are permitted.

---

## **2. APM Termux Runtime Root**

All Termux packages installed by APM MUST live inside:

```
/data/apm/termux/
    usr/
    bin/
    lib/
    share/
    home/
    tmp/
    env.sh
```

### Directories APM MUST create:

```
/data/apm/termux
/data/apm/termux/usr
/data/apm/termux/usr/bin
/data/apm/termux/usr/lib
/data/apm/termux/usr/share
/data/apm/termux/tmp
/data/apm/termux/home
```

These are the Termux equivalents of the standard Termux filesystem.

---

## **3. Termux Repo Detection**

A repository is considered Termux if ANY of these are true:

1. The repo URL contains:

   ```
   packages.termux.dev
   ```
2. A `Packages` entry filename includes:

   ```
   data/data/com.termux/files/usr
   ```
3. A package entry contains Termux-specific dependencies such as:

   * `libandroid-glob`
   * `libandroid-support`
   * `termux-tools`
   * etc.

### Implementation:

Extend `RepoSource` with:

```
bool isTermuxRepo = false;
```

When parsing sources.list and Packages files, set this flag.

---

## **4. Termux Package Detection**

When extracting a `.deb`, APM MUST detect Termux package layout:

```
data/data/com.termux/files/usr/...
```

If this structure exists, set:

```
InstallOptions.isTermuxPackage = true;
```

This is used by:

* Install manager
* Dependency resolver
* Wrapper generator
* Status DB

---

## **5. Path Remapping (Critical)**

Termux packages install under:

```
data/data/com.termux/files/usr/bin/<binary>
data/data/com.termux/files/usr/lib/<lib>
data/data/com.termux/files/usr/share/<...>
```

APM MUST remap these to:

```
/data/apm/termux/usr/bin/
/data/apm/termux/usr/lib/
/data/apm/termux/usr/share/
```

### Required behaviors:

* During extraction, rewrite all internal paths
* Preserve directory structure below `/usr`
* The original parent directories must not be kept

---

## **6. Termux Environment File**

Create:

```
/data/apm/termux/env.sh
```

Contents MUST include:

```
export PREFIX=/data/apm/termux/usr
export HOME=/data/apm/termux/home
export PATH=$PREFIX/bin:$PATH
export LD_LIBRARY_PATH=$PREFIX/lib
export TMPDIR=/data/apm/termux/tmp
export TERM=xterm-256color
export LANG=en_US.UTF-8
export SHELL=/system/bin/sh
```

APM MUST ensure `env.sh` exists before using it.

---

## **7. Binary Wrapper Scripts**

For each Termux binary installed to:

```
/data/apm/termux/usr/bin/<command>
```

APM MUST generate a wrapper script:

Path:

```
/data/apm/bin/<command>
```

Contents:

```
#!/system/bin/sh
. /data/apm/termux/env.sh
exec /data/apm/termux/usr/bin/<command> "$@"
```

### Additional requirements:

* Must be executable (`chmod 0755`)
* If a wrapper already exists, overwrite it

---

## **8. Termux Dependency Resolution**

Termux package dependencies work differently from Debian.

### Rules for Termux-mode dependency resolution:

1. Only resolve dependencies within Termux repos
2. Ignore version constraints entirely
3. Ignore architecture mismatches
4. Do not attempt to resolve Debian/system packages
5. If a dependency is missing in Termux repos, mark as missing but continue if user chooses `--force` in the future

### Implementation:

Add a new dependency resolver path:

```
resolveTermuxDependencies()
```

Called when:

```
InstallOptions.isTermuxPackage == true
```

---

## **9. Status DB Support**

Add **two new fields** when saving a Termux-installed package:

```
Termux-Package: yes
Install-Prefix: /data/apm/termux/usr
```

These must appear in status entries the same way other fields are recorded.

---

## **10. Removal of Termux Packages**

When uninstalling a Termux package, APM MUST:

1. Remove:

   ```
   /data/apm/termux/usr/bin/<binary>
   /data/apm/termux/usr/lib/<libs>
   /data/apm/termux/usr/share/<data>
   ```
2. Remove wrapper:

   ```
   /data/apm/bin/<command>
   ```
3. Update status DB
4. Remove empty directories under `/data/apm/termux/usr`
5. Leave `/data/apm/termux/env.sh` intact
6. Leave `/data/apm/termux/home` intact

---

## **11. No Interaction With Real Termux App**

APM MUST NOT:

* Write to `/data/data/com.termux/`
* Modify the real Termux installation
* Create or modify Termux app files

All Termux functionality must remain inside `/data/apm/termux`.

---

## **12. Debian Packages and Termux Packages Must Not Intermix**

The install root for Termux is:

```
/data/apm/termux
```

The install root for Debian-style packages is:

```
/data/apm/pkgs
```

APM MUST NOT:

* Allow Debian packages to depend on Termux packages
* Allow Termux packages to depend on Debian packages

These are separate systems.

---

## **13. APMD Support**

APMD must:

* Detect `isTermuxPackage` installation
* Forward Termux install instructions to installation manager
* Handle Termux uninstall
* Log Termux actions separately from Debian actions

No additional daemon-level changes required.

---

## **14. Summary of Required Additions**

### New flags

```
RepoSource.isTermuxRepo
InstallOptions.isTermuxPackage
InstalledPackage.TermuxPackage
InstalledPackage.InstallPrefix
```

### New directories under APM root

```
/data/apm/termux/
/data/apm/termux/usr/
```

### New functions needed

* `detectTermuxRepo(...)`
* `detectTermuxPackage(...)`
* `rewriteTermuxPathsDuringExtraction(...)`
* `createTermuxEnvFileIfMissing()`
* `createTermuxWrapper(commandName)`
* `resolveTermuxDependencies(...)`
* `removeTermuxPackageFiles(...)`

---

## **15. Future Extensions (Not required now)**

These are optional and must not be implemented yet:

* Patching ELF interpreter using `patchelf`
* Emulating `/data/data/com.termux/files/home`
* termux-exec integration
* Shell profile integration
* Termux package scripts (`postinst`, etc.)
